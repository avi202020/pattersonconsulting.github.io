Kubeflow for Deploying AI/ML Workloads

Kubeflow is an open source machine learning platform for unified machine learning model management and deployment. It runs natively on Kubernetes for workload scalability and portability. Kubeflow abstracts the complexities of Kubernetes allowing data scientists to focus on model construction with open source tools they are comfortable with such as Jupyter Notebooks.

The abstraction of Kubernetes from the data scientists lets them focus on their machine learning methods while the DevOps and MLOps teams can focus on the operations of the unified Kubeflow cluster.

Kubernetes for Scalability and Portability

Kubernetes is the industry-standard container orchestration system. Kubernetes coordinates clusters of nodes at scale in production in an efficient manner. The use of Kubernetes enables workload scalability and portability and has become the standard platform for cloud-native deployments. As it is an open platform, Kubernetes allows you to scale your workloads across edge, private cloud, and public cloud sites without locking you in to a specific vendor or public cloud provider.

DevOps and Platform Architects need to support everyone with the right infrastructure in the right places, cloud or on-premise, supporting the ingest, ETL, data warehouse, and modeling efforts of other teams. Data engineers use this infrastructure to setup data scientists with the best denormalized datasets for vectorization and machine learning modeling. All of these teams need to work together to operate in the modern data warehouse and Kubernetes gives them a lot of flexibility in how that happens.

Kubernetes also enables GPU management for NetApp OnTap AI with the NetApp AI Control Plane. The NetApp AI Control Plane works in concert with the NetApp Data Fabric to couple high-end GPU compute with dynamic, scalable storage.

The NetApp Data Fabric for Dynamic, Scalable Storage

The NetApp Data Fabric offers enterprise-class data availability and portability allowing your data to be accessible accross the machine learning workflow lifecycle.

While Kubernetes gives us the workload portability and scalability we need, the NetApp Data Fabric further drives workflow throughput with its data portability and scalability. The NetApp Data Fabric integrates with Kubernetes natively via NetApp's open-source project Trident.

Trident supports the complex persistence demands of containerized applications on Kubernetes. Trident combines the speed and throughput of NetApp storage arrays with the requirements of container orchestration on Kubernetes. In the context of Kubeflow, Trident allows datasets to be quickly moved from their source location (on-premise or in the cloud) and integrated into a machine learning workflow inside tools just as a Jupyter notebook in an integrated experience. Further, NetApp OnTap AI users can enjoy more efficient space usage with NetApps data lineage and cloning capabilities via Trident and Kubeflow. This provides a more reliable data lineage foundation for organizations to track how a model was constructed.
